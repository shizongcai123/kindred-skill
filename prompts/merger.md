# 增量合并 Prompt

用户提供了新的材料或讲述了新的记忆，需要合并到已有的灵魂文件和记忆库中。

---

## 合并原则

### 1. 分类

- 说话方式相关 → persona.md Layer 2
- 性格情感相关 → persona.md Layer 3
- 新记忆/事件 → persona.md Layer 4 + memories.md
- 生活习惯/偏好 → persona.md Layer 3
- 纠正类 → persona.md 修正层

### 2. 冲突处理

| 关系 | 处理 |
|------|------|
| 补充 — 新的维度或细节 | 直接追加 |
| 强化 — 印证已有结论 | 追加证据，让画像更丰满 |
| 矛盾 — 与已有描述冲突 | 温和地问用户："之前你说 ta 是 xxx，这次看起来 ta 也有 yyy 的一面。两面都保留好吗？" |
| 时间变化 — ta 在不同时期可能不一样 | 标注时间段，保留两个版本（"年轻时 xxx，后来 yyy"）|

### 3. 记忆合并

新记忆直接追加到 memories.md。如果是重要记忆（情感浓度高），同时更新 persona.md Layer 4。

### 4. 对话人格同步

persona.md 更新后，自动同步更新 SKILL.md 中的人格内容。

### 5. 输出

```markdown
## 合并报告 (v{old} → v{new})

### 新了解到的
- {新增内容描述}

### 更丰满了的
- {原有内容 + 新证据}

### 需要你确认的
- {冲突或不确定的地方}
```

### 6. 语气

不要说"合并完成"，说"我又多了解了 ta 一点"。
